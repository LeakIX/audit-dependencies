## Overview

Pure Rust implementation of the AES-GCM (Galois/Counter Mode) Authenticated
Encryption with Associated Data (AEAD) Cipher with optional
architecture-specific hardware acceleration.

## Known Vulnerabilities

### CVE-2023-42811 / RUSTSEC-2023-0096 / GHSA-423w-p2w9-r7vq

| Field | Value |
|-------|-------|
| **Status** | FIXED in v0.10.3 |
| **Severity** | MEDIUM |
| **CVSS (NIST)** | 5.5 (CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:N/A:N) |
| **CVSS (GitHub)** | 4.7 (CVSS:3.1/AV:L/AC:H/PR:H/UI:N/S:U/C:L/I:H/A:N) |
| **CWE** | CWE-347 (Improper Verification of Cryptographic Signature) |
| **Affected Versions** | >= 0.10.0, <= 0.10.2 |
| **Patched Version** | 0.10.3 |
| **Published** | September 21, 2023 |

#### Description

The `decrypt_in_place_detached` function in aes-gcm versions 0.10.0 through
0.10.2 exposes decrypted plaintext even when authentication tag verification
fails. This violates NIST GCM guidelines which specify that plaintext should
only be returned after successful tag verification.

#### Technical Details

- Decryption occurs in-place before authentication tag validation
- On tag verification failure, the buffer contains unauthenticated plaintext
- This enables potential Chosen Ciphertext Attacks (CCAs)
- Could lead to full plaintext recovery depending on application behavior

#### Affected Functions

- `aes_gcm::AeadInPlace::decrypt_in_place_detached`
- `aes_gcm::AesGcm::decrypt_in_place_detached`

#### Fix

Version 0.10.3 implements proper plaintext protection by re-encrypting the
decrypted buffer before returning errors, consistent with NIST guidelines.

## Dependencies

The crate has minimal dependencies and supports optional features:

- `aes` (default) - AES cipher implementation
- `alloc` (default) - Alloc support
- `getrandom` (default) - Random number generation
- `rand_core` - Random core traits
- `arrayvec`, `bytes`, `heapless` - Optional buffer types

## Recommendations

1. **Use version 0.10.3 or later** - This is the patched version
2. **Do not use versions 0.10.0-0.10.2** - These contain CVE-2023-42811
3. **Avoid pre-release versions** in production (0.11.0-rc.x)
4. **Monitor RustSec** for future advisories
