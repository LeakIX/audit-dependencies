## Crate Overview

Tower is a library of modular and reusable components for building robust
clients and servers. It provides a protocol-agnostic request/response pattern
for building networking applications.

### Features

The library offers modular components including:

- balance, buffer, discover, filter, hedge
- limit, load, load-shed, make, ready-cache
- reconnect, retry, spawn-ready, steer, timeout
- utility functions

A "full" feature flag enables all components.

### Related Crate Advisories

Note: The related crate `tower-http` (a separate crate providing HTTP-specific
Tower utilities) has had security advisories:

- **RUSTSEC-2022-0043**: Directory traversal vulnerability in
  `ServeDir` on Windows platforms (fixed in tower-http 0.1.3 and 0.2.1)

This advisory does NOT affect the core `tower` crate being audited here.

## Recommendations

1. **Safe to use** - No known security vulnerabilities in tower v0.5.3
2. **Monitor dependencies** - If using `tower-http`, ensure version >= 0.2.1
3. **Enable cargo-audit** - Run `cargo audit` in CI to catch future advisories
4. **Pin versions** - Consider pinning to specific versions in production
