developers to build command-line parsers declaratively or procedurally with all
standard CLI features.

### CVE Records

**Status: No known CVEs**

No CVE records were found directly associated with the `clap` crate.

### Historical Indirect Vulnerabilities

Older versions of `clap` (2.x series) had indirect vulnerabilities through
transitive dependencies:

- **regex crate vulnerability**: The Rust Security Response WG identified that
  the regex crate did not properly limit complexity of parsed regular
  expressions, allowing denial of service attacks. This affected older clap
  versions that depended on vulnerable regex versions.

**Mitigation:** Upgrade to clap 4.x which uses updated dependencies.
