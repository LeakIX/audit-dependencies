## Related Security Incidents

### CVE-2025-54313 (NOT in prettier core)

A supply chain attack affected related ESLint-Prettier tooling packages, **not the
prettier package itself**:

- **Affected Packages:** `eslint-config-prettier`, `eslint-plugin-prettier`,
  `synckit`, `@pkgr/core`, `napi-postinstall`
- **Severity:** High (CVSS 7.5)
- **Impact:** Windows-only remote code execution payload
- **Root Cause:** Maintainer phishing attack via spoofed npm email

**Important:** The core `prettier` package was NOT affected by this incident.

## Package Integrity

The package is published with:

- SHA-512 integrity hash verification
- Signed with SHA256 signature by trusted publisher (GitHub Actions OIDC)
- OIDC-based trusted publisher authentication

## Dependencies

Prettier has **zero runtime dependencies**, which significantly reduces supply chain
attack surface.

## Recommendations

1. **Safe to use** - No known vulnerabilities in prettier v3.8.1
2. **Monitor related packages** - If using `eslint-config-prettier` or
   `eslint-plugin-prettier`, ensure you have patched versions (8.10.2+, 9.1.2+,
   10.1.8+)
3. **Use lockfiles** - Pin exact versions in package-lock.json
4. **Enable npm audit** - Run `npm audit` regularly in CI

## Conclusion

Prettier v3.8.1 has no known security vulnerabilities. The package has zero runtime
dependencies and is published through GitHub Actions OIDC trusted publishing, making
it a low-risk dependency from a security perspective.
