# Security Audit: react-router-dom v7.13.0

**Package:** [react-router-dom](https://www.npmjs.com/package/react-router-dom)
**Version:** 7.13.0
**Audit Date:** 2026-02-01
**License:** MIT

## Package Information

- **Description:** DOM bindings for React Router
- **Repository:** https://github.com/remix-run/react-router
- **Homepage:** https://reactrouter.com/
- **Maintainers:** mjackson, timdorr (React Training / Remix team)

## Security Status

**Overall Assessment:** SECURE (v7.13.0)

The `react-router-dom` package itself has no direct vulnerabilities in Snyk's
database. However, it depends on `react-router`, which has had multiple
high-severity vulnerabilities in recent versions. Users of **Framework Mode**
should ensure they are running v7.12.0 or later.

**Important:** Applications using **Declarative Mode** (`<BrowserRouter>`) or
**Data Mode** (`createBrowserRouter`/`<RouterProvider>`) are generally NOT
affected by most of these vulnerabilities.

## Known Vulnerabilities (via react-router dependency)

### Critical: CVE-2025-61686 - Directory Traversal

| Field              | Value                                                    |
| ------------------ | -------------------------------------------------------- |
| **CVE**            | CVE-2025-61686                                           |
| **CVSS**           | 9.1 (Critical)                                           |
| **Type**           | Path Traversal (CWE-22)                                  |
| **Affected**       | @react-router/node 7.0.0 - 7.9.3                         |
| **Fixed Versions** | @react-router/node 7.9.4, @remix-run/node 2.17.2         |
| **Disclosed**      | 2025-12                                                  |

**Description:** The `createFileSessionStorage()` function in @react-router/node
is vulnerable to path traversal when used with unsigned cookies. An attacker can
craft a malicious session ID containing `../` sequences to read or write files
outside the designated session directory.

**Impact:**

- Read files matching session file format specifications
- Modify session data returned by application logic
- Potentially access sensitive configuration files

**Advisory:**
[GHSA-9583-h5hc-x8cw](https://github.com/advisories/GHSA-9583-h5hc-x8cw)

### High: CVE-2026-22029 - XSS via Open Redirects

| Field              | Value                                                 |
| ------------------ | ----------------------------------------------------- |
| **CVE**            | CVE-2026-22029                                        |
| **CVSS**           | 8.0 (High)                                            |
| **Type**           | Cross-Site Scripting via Open Redirect                |
| **Affected**       | react-router 7.0.0 - 7.11.0, @remix-run/router <=1.23.1 |
| **Fixed Versions** | react-router 7.12.0, @remix-run/router 1.23.2         |
| **Disclosed**      | 2026-01                                               |

**Description:** SPA open navigation redirects originating from loaders or
actions in Framework Mode, Data Mode, and unstable RSC modes can result in
unsafe URLs causing unintended JavaScript execution on the client.

**Impact:**

- Arbitrary JavaScript execution in user's browser
- Session hijacking
- Data theft

**Advisory:**
[GHSA-2w69-qvjg-hvjx](https://github.com/advisories/GHSA-2w69-qvjg-hvjx)

### High: CVE-2025-43865 - Pre-render Data Spoofing

| Field              | Value                                      |
| ------------------ | ------------------------------------------ |
| **CVE**            | CVE-2025-43865                             |
| **CVSS**           | 8.2 (High)                                 |
| **Type**           | Data Tampering / Cache Poisoning           |
| **Affected**       | react-router 7.0.0-pre.0 - 7.5.1           |
| **Fixed Versions** | react-router 7.5.2                         |
| **Disclosed**      | 2025-04                                    |

**Description:** Attackers can modify pre-rendered data by adding the
`X-React-Router-Prerender-Data` header to requests. This allows complete content
spoofing and modification of all data object values passed to HTML.

**Impact:**

- Cache poisoning attacks
- Stored XSS vulnerabilities
- Complete content manipulation via loader data alteration

**Advisory:**
[GHSA-cpj6-fhp6-mr6j](https://github.com/advisories/GHSA-cpj6-fhp6-mr6j)

### High: CVE-2025-59057 - XSS in meta()/Meta APIs

| Field              | Value                                        |
| ------------------ | -------------------------------------------- |
| **CVE**            | CVE-2025-59057                               |
| **CVSS**           | 7.6 (High)                                   |
| **Type**           | Cross-Site Scripting                         |
| **Affected**       | react-router 7.0.0 - 7.8.2                   |
| **Fixed Versions** | react-router 7.9.0, @remix-run/react 2.17.1  |
| **Disclosed**      | 2026-01-08                                   |

**Description:** XSS vulnerability in React Router's `meta()`/`<Meta>` APIs in
Framework Mode when generating `script:ld+json` tags. Arbitrary JavaScript can
execute during SSR if untrusted content is used to generate the tag.

**Impact:**

- Arbitrary JavaScript execution during server-side rendering
- Data exfiltration
- Session hijacking

**Advisory:**
[GHSA-3cgp-3xvw-98x8](https://github.com/advisories/GHSA-3cgp-3xvw-98x8)

### Medium: CVE-2026-22030 - Server Action CSRF

| Field              | Value                                                 |
| ------------------ | ----------------------------------------------------- |
| **CVE**            | CVE-2026-22030                                        |
| **CVSS**           | 6.5 (Medium)                                          |
| **Type**           | Cross-Site Request Forgery                            |
| **Affected**       | react-router 7.0.0 - 7.11.0, @remix-run/server-runtime <2.17.3 |
| **Fixed Versions** | react-router 7.12.0, @remix-run/server-runtime 2.17.3 |
| **Disclosed**      | 2026-01                                               |

**Description:** React Router is vulnerable to CSRF attacks on document POST
requests to UI routes when using server-side route action handlers in Framework
Mode, or when using React Server Actions in unstable RSC modes.

**Impact:**

- Unauthorized actions performed on behalf of authenticated users
- State manipulation
- Data modification

### Medium: CVE-2025-68470 - External Redirect

| Field              | Value                                      |
| ------------------ | ------------------------------------------ |
| **CVE**            | CVE-2025-68470                             |
| **CVSS**           | 6.5 (Medium)                               |
| **Type**           | Open Redirect                              |
| **Affected**       | react-router (various versions)            |
| **Fixed Versions** | react-router 7.12.0                        |
| **Disclosed**      | 2025-12                                    |

**Description:** An attacker-supplied path can be crafted so that when a React
Router application navigates to it via `navigate()`, `<Link>`, or `redirect()`,
the app performs a navigation/redirect to an external URL.

**Note:** This vulnerability has been exploited in the wild.

**Impact:**

- Phishing attacks via trusted domain redirects
- Credential theft

## Who Is Affected

You are affected if your application:

- Uses **Framework Mode** (the full React Router framework with SSR)
- Uses **Data Mode** (`createBrowserRouter`/`<RouterProvider>`) with loaders
- Uses `@react-router/node` with `createFileSessionStorage()`
- Uses server-side route action handlers
- Uses React Server Actions in unstable RSC modes

## Who Is NOT Affected

You are NOT affected if:

- You use **Declarative Mode** (`<BrowserRouter>`, `<HashRouter>`) without
  loaders/actions
- Your React app is purely client-side rendered
- You are using versions 7.12.0+ of react-router
- You are using versions prior to 7.0.0 (v5.x, v6.x lines)

## Remediation

### Upgrade to Latest Version

```bash
npm install react-router-dom@7.13.0
```

This automatically pulls in react-router@7.13.0 which includes all security
patches.

### For Remix Users

```bash
npm install @remix-run/react@2.17.3 @remix-run/router@1.23.2
npm install @remix-run/node@2.17.2  # If using file session storage
```
