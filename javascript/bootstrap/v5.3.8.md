# Security Audit: bootstrap v5.3.8

**Package:** [bootstrap](https://www.npmjs.com/package/bootstrap)
**Version:** 5.3.8
**Audit Date:** 2026-02-01
**License:** MIT

## Package Information

- **Description:** The most popular front-end framework for developing responsive,
  mobile first projects on the web
- **Repository:** https://github.com/twbs/bootstrap
- **Homepage:** https://getbootstrap.com/
- **Maintainers:** Twitter Bootstrap Core Team

## Security Status

**Overall Assessment:** SECURE

Bootstrap 5.3.8 has no known direct vulnerabilities according to Snyk, NVD, and
GitHub Advisory databases. All historical XSS vulnerabilities affect only
Bootstrap 3.x and 4.x (both end-of-life).

## Known Vulnerabilities

### Bootstrap 5.x: No Known CVEs

Bootstrap 5.3.8 is the latest version and has no published CVEs or security
advisories in the official GitHub repository.

### Historical Vulnerabilities (Older Versions Only)

The following vulnerabilities affect **only Bootstrap 3.x and 4.x** and do NOT
affect Bootstrap 5.x:

#### CVE-2025-1647 - XSS in Tooltip/Popover (Bootstrap 3 Only)

| Field | Value |
|-------|-------|

| **CVSS** | 5.6 (Medium) |
| **Type** | Cross-Site Scripting (XSS) |

| **Fixed Versions** | 4.0.0+ (Bootstrap 3 is EOL) |

| Field | Value |

## Bootstrap Security Model

Bootstrap maintainers have clarified their security position:

> "Bootstrap's JavaScript is not intended to sanitize unsafe or intentionally
> dangerous HTML."

This means developers are responsible for sanitizing user input before passing
it to Bootstrap components. Applications should:

1. **Never** pass untrusted user input directly to Bootstrap component attributes
2. Sanitize all user-controlled data before rendering in HTML
3. Use Content Security Policy (CSP) headers
4. Validate and escape data on the server side

## Supply Chain Considerations

### ngx-bootstrap Malware Incident (September 2025)

On September 15, 2025, malicious versions of `ngx-bootstrap` (an Angular wrapper
for Bootstrap, **not** the core Bootstrap package) were published to npm. These
contained a postinstall script that harvested developer tokens and credentials.

**This does NOT affect the core `bootstrap` npm package.**

Affected package: `ngx-bootstrap` (not `bootstrap`)
Advisory: GHSA-6m4g-vm7c-f8w6

## Remediation

### For Users of Bootstrap 5.3.8

No action required. This version has no known vulnerabilities.

### For Users of Bootstrap 3.x or 4.x

**Immediate action required:** Migrate to Bootstrap 5.x

```bash
npm install bootstrap@5.3.8
```

### Security Best Practices

1. Always sanitize user input before passing to Bootstrap components
2. Use Content Security Policy (CSP) headers
3. Keep Bootstrap updated to the latest version
4. Do not use deprecated Bootstrap 3.x or 4.x versions
