## Key Features

- Load `.env` files automatically in the same directory as Python scripts
- Parse configuration without modifying the environment using `dotenv_values()`
- Support for variable expansion with POSIX-style syntax
- Command-line interface for manipulating `.env` files
- Multiline value support with escape sequences
- Optional disable via `PYTHON_DOTENV_DISABLED` environment variable

## Security Best Practices

When using `python-dotenv`:

1. **Never commit `.env` files** - Add `.env` to `.gitignore`
2. **Use `.env.example`** - Provide a template without actual secrets
3. **Restrict file permissions** - Use `chmod 600 .env` on production systems
4. **Validate environment variables** - Don't trust values blindly
5. **Use different `.env` files per environment** - Separate dev/staging/prod
6. **Consider encryption** - For highly sensitive values, use a secrets manager
