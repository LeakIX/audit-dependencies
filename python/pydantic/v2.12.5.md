fast and extensible data validation with excellent IDE/linter support.

## Known Vulnerabilities

### CVE-2024-3772 - Regular Expression Denial of Service (ReDoS)

| Field | Value |
|-------|-------|
| **CVE ID** | CVE-2024-3772 |
| **Severity** | HIGH (NIST) / MEDIUM (CNA) |
| **CVSS v3.1** | 7.5 (NIST) / 5.9 (CNA) |
| **Vector** | AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:H |
| **CWE** | CWE-1333 (Inefficient Regular Expression Complexity) |
| **Published** | April 15, 2024 |
| **Affected** | < 1.10.13, >= 2.0.0 < 2.4.0 |
| **Fixed In** | 1.10.13, 2.4.0 |

**Description:**

Regular expression denial of service in Pydantic allows remote attackers to
cause denial of service via a crafted email string. The vulnerability exists in
the email validation mechanism where the regular expression pattern used for
validation could be exploited through carefully crafted input containing
excessive whitespace characters.

**Impact:**

When exploited, this vulnerability can cause a denial of service condition by
consuming excessive CPU resources during email validation, potentially making
the affected service unresponsive.

**References:**

- [NVD Entry](https://nvd.nist.gov/vuln/detail/CVE-2024-3772)
- [GitHub Advisory GHSA-mr82-8j83-vxmv](https://github.com/advisories/GHSA-mr82-8j83-vxmv)
- [Snyk Advisory](https://security.snyk.io/vuln/SNYK-PYTHON-PYDANTIC-5907722)
- [GitHub PR #7360](https://github.com/pydantic/pydantic/pull/7360)

### CVE-2021-29510 - Denial of Service via Infinity Values

| Field | Value |
|-------|-------|
| **CVE ID** | CVE-2021-29510 |
| **Severity** | HIGH (NIST) / LOW (CNA) |
| **CVSS v3.1** | 7.5 (NIST) / 3.3 (CNA) |
| **Vector** | AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:H |
| **CWE** | CWE-835 (Infinite Loop with Unreachable Exit Condition) |
| **Published** | May 13, 2021 |
| **Affected** | < 1.6.2, < 1.7.4, < 1.8.2 |
| **Fixed In** | 1.6.2, 1.7.4, 1.8.2 |

**Description:**

Passing infinity values (`'infinity'`, `'inf'`, or `float('inf')`) to datetime
or date fields in Pydantic causes the validation process to hang indefinitely
while consuming 100% CPU resources.

**Impact:**

Applications accepting user input for datetime/date fields could be crashed or
made unresponsive by malicious input containing infinity values.

**References:**

- [NVD Entry](https://nvd.nist.gov/vuln/detail/CVE-2021-29510)
- [GitHub Advisory GHSA-5jqp-qgf6-3pvh](https://github.com/pydantic/pydantic/security/advisories/GHSA-5jqp-qgf6-3pvh)
- [Snyk Advisory](https://security.snyk.io/vuln/SNYK-PYTHON-PYDANTIC-1292338)

## Related Vulnerabilities (Not in Pydantic Core)

### CVE-2025-22151 - Strawberry GraphQL Type Confusion

This vulnerability affects Strawberry GraphQL's relay integration, not pydantic
directly. Pydantic is one of the affected ORM integrations where the
vulnerability manifests, but the fix is in Strawberry GraphQL 0.257.0.

- **Severity:** LOW (CVSS 3.7)
- **Affected:** Strawberry GraphQL 0.182.0 - 0.256.x
- **Fixed In:** Strawberry GraphQL 0.257.0

## Security Policy

Pydantic has a security policy for responsible disclosure:

- **Do not** open public issues for security vulnerabilities
- Report via GitHub's "Report a vulnerability" feature in the Security tab
- Include detailed information with a minimal reproducible example

**Policy URL:** https://github.com/pydantic/pydantic/security/policy

**Note:** The security policy does not specify response timeframes or
coordinated disclosure timelines.

## Recommendations

1. **Use version 2.12.5 or later** - All known vulnerabilities are patched
2. **Monitor for updates** - Subscribe to GitHub security advisories
3. **Input validation** - Even with pydantic, validate untrusted input at
   application boundaries
4. **Rate limiting** - Implement rate limiting for endpoints accepting user
   input to mitigate potential DoS attacks
